Javascript Fundamentals

What is HTML, CSS and Javascript?

HTML provides the content and the structure of a webpage. Titles, paragraphs, images etc.

CSS adds the style to a webpage, and makes it look good. It changes the color of text, the size of an image etc.

CSS needs HTML, otherwise there will be no content to make pretty/ugly for the webpage.

Javascript allows interaction and magic to a webpage.

An analogy would be a birthday card.

We can think of HTML as the words on the front and the middle of the page i.e “Happy birthday” on the front cover with a birthday cake. Inside a message “Happy birthday to my most boring friend.”

The CSS would be selecting how it looks - the font, the layout, how big or small the image of the birthday cake on the front is.

Javascript is adding an action or interactive element to the birthday card. In this case, when the birthday card is opened, a photo of your boring friend asleep pops out, along with the song “happy birthday” played at half speed.

What is control flow? Why does it relate to cake?

Control flow is the order a computer will read, interpret and execute statements in a script. Typically, code is run in order from top to bottom, this is the flow.

There are a few things which may change the control flow including conditionals, loops and functions.

What is the DOM? How can I interact with it? Will she be gentle?

The DOM is the Document Object Model, it is a programming interface which allows programs to read and manipulate a page’s content. It acts as a set of instructions for web browsers.

It can be represented hierarchically using a tree of nodes or elements by the browser. It all actually starts with the window - the matriarch of our family. The window is not actually a ‘true’ element of the DOM, so she is not listed on the diagram. It is more your web browser. But just know that she is there, above us all.

Underneath her, is Document. This is the representation of a webpage. Below her we have our root element - our html. Next, the family starts having more children - we have two child elements - a head and a body. They have their own children, and then their children have their own children…and so on.

…Something to remember though is that there can only ever be one parent of a child element. It’s a good thing. This helps us find specific elements easier.

Say we had an html button with an id attribute of hotStuff. We could use Javascript to find this hotStuff button on our document (our webpage), and modify it, perhaps make it a smoking red color each time it is clicked using an event listener. Maybe at some point a capable programmer could make the pressing of the button play the chorus of Donna Summer’s 1979 smash hit “Hot Suff” for an unsuspecting webpage user.

Objects? Arrays? Explain the difference between accessing data from each.

We can use both objects and arrays to store data, but they are useful for different data types.

A javascript object is a collection of data. It has a list of properties, each with a name(key), which has assigned to it a value

A dictionary is a good metaphor here. An object is a dictionary. It has a number of names of words(referred to as keys), and the definition of the word (meaning the value). The pair is known as the key-value pair (or…the word-definition pair)

Here is an example of a javascript object.

let hotStuff = {

firstName: ‘Donna’,

lastName: ‘Summer”,

favouriteSeason: ‘Winter’,

To access the data, you want to use dot notation or bracket notation. We can use dot notation to pull out hotStuff.firstName to give us Donna.

Arrays on the other hand: Are a list of things. You can enter multiple items inside one variable. The list is indexed - every item in the list in an array has an index number, and we can then call an item from the array list, by calling a number. AN INTERESTING AND CONFUSING THING TO NOTE is that in programming, arrays start at zero. So when counting, we count 0,1,2,4 etc.
